## 阶段 2：数据库 + 剪贴板监听核心 ✅ 已完成

**目标**：实现最小可用核心——能记录剪贴板内容到数据库。

### 步骤
1. ✅ 创建 `src/` 文件夹和基础模块文件
2. ✅ 编写 `src/database.py`：建表 + 增删查操作
3. ✅ 编写 `src/clipboard_monitor.py`：监听剪贴板，识别文字/图片，去重，写入 DB
4. ✅ 编写 `src/main.py` 入口骨架（仅启动监听 + 打印日志，无 UI）
5. ✅ 编写 `requirements.txt`
6. ✅ 安装依赖并手动测试

### 关键决策
- 确认 Python 环境就绪
- 确认 PySide6 剪贴板 API 可用
- 确认 SQLite 读写正常

### 验收标准
- 复制文字 → 数据库中新增一条 type='text' 的记录
- 复制图片 → 数据库中新增一条 type='image' 的记录，图片文件保存到本地
- 连续复制相同内容 → 不产生重复记录
- 手动检查数据库能查到记录

---

## 阶段 3：系统托盘 + 主窗口框架 ✅ 已完成

**目标**：软件有可见的交互入口——托盘图标和主窗口。

### 步骤
1. 完善 `src/main.py`：创建 QApplication + 系统托盘
2. 编写 `src/ui/main_window.py`：空窗口框架（带搜索栏占位、底部状态栏）
3. 应用样式表 `resources/style.qss`
4. 托盘菜单：打开 / 退出

### 验收标准
- 运行后系统托盘出现图标
- 点击托盘图标 → 打开/关闭主窗口
- 右键托盘 → 显示"打开"和"退出"菜单
- 窗口有淡粉色背景，符合设计规范

---

## 阶段 4：卡片组件 + 搜索 ✅ 已完成

**目标**：主窗口显示历史记录卡片，支持搜索。

### 步骤
1. 编写 `src/ui/card_widget.py`：文字卡片 + 图片卡片
2. 主窗口从数据库加载记录，渲染卡片列表
3. 实现搜索栏实时过滤（300ms 防抖）
4. 实现点击卡片 → 复制到剪贴板
5. 实现卡片新增/删除时的列表刷新

### 验收标准
- 打开主窗口 → 看到所有历史记录卡片（时间降序）
- 输入搜索 → 卡片列表实时过滤
- 点击文字卡片 → 内容复制到剪贴板，可粘贴验证
- 点击图片卡片 → 图片复制到剪贴板，可粘贴验证
- 新复制内容后打开面板 → 新卡片出现在顶部

---

## 阶段 5：设置面板 + 置顶/删除 + 轻提示 ✅ 已完成

**目标**：完善交互功能——置顶、删除、保留天数设置、轻提示。

### 步骤
1. 编写 `src/ui/settings_dialog.py`：保留天数设置
2. 编写 `src/utils/config.py`：配置读写（JSON 文件）
3. 实现卡片右键菜单：复制 / 置顶 / 删除
4. 实现删除确认对话框
5. 编写 `src/ui/toast.py`：轻提示组件
6. 编写 `src/utils/cleanup.py`：过期清理逻辑 + 定时器

### 验收标准
- 右键卡片 → 可置顶，置顶卡片始终在顶部
- 右键卡片 → 可删除，确认后卡片消失
- 设置中可切换 1/3/5 天保留期
- 复制内容后右下角出现轻提示
- 过期记录自动清理
